## English

# Developer Experience Observatory

An open, neutral observatory for measuring  
unexpected time costs in real-world software development.

---

## Purpose

**Developer Experience Observatory** is a neutral observatory that collects
and analyzes **unexpected time costs** experienced by developers
in real-world software development environments.

The goal of this project is not to evaluate products or organizations,
but to **observe structural patterns** that affect developer time,
learning cost, and productivity at scale.

Over time, this work aims to help reduce incidental time costs — such as
re-learning user interfaces or responding to frequent specification changes —
that do not directly contribute to value creation.

Such observations matter because unexpected volatility in development environments
is associated with significant, often invisible economic losses over time.

---

## What This Project Is / Is Not

### What this project is

- An **observational data project**
- A place to record **unexpected time loss** caused by:
  - unannounced or unclear changes
  - documentation lag
  - tooling inconsistencies
- A shared data source to support **constructive discussion**
  on improving developer experience

### What this project is NOT

- A bug tracker
- A customer support or escalation channel
- A ranking or scoring system for products or organizations
- A platform for blaming, attacking, or targeting individuals,
  teams, companies, or products

---

## Privacy & Anonymity

- All submissions are **anonymous**
- No account or social login is required
- No personal identifiers are intentionally collected
- Submissions are protected against automated abuse
  (e.g., bot submissions)

This project is designed to **collect as little data as possible**
while retaining meaningful analytical value.

---

## Data Collection Policy

### What is collected

Each submission records only high-level information such as:

- Platform (e.g. iOS, Web, Android)
- Tool or service category
- Type of change (e.g. announced / unannounced / unclear)
- Approximate time lost (range-based)
- Team size (range-based)

Optional free-text descriptions are limited in length
and used only to support aggregation and qualitative understanding.

### What is NOT collected

- Names of individuals
- Account identifiers or login information
- Internal company details
- Source code or proprietary data

---

## Data Limitations and Scope

This project does **not verify the accuracy of individual submissions**.
All reports are treated as **subjective observations** by developers.

The collected data is **not intended to evaluate, rank, or judge**
specific companies, products, or engineering organizations.
It is intended solely to observe patterns of
unexpected time costs in development work.

Observed correlations do **not imply causation**.
Any structural interpretations are presented as hypotheses,
not conclusions.

---

## Data Publication

- Raw individual submissions are **not published**
- Aggregated data is published periodically (e.g. monthly)
- Published formats include CSV files and visual summaries
- Methodology and analysis code are fully open

Individual submissions may be referenced in anonymized,
non-attributable form (e.g., *"A developer reported..."*)
**solely for illustrative purposes**.

Such references are **not presented as verified facts**,
but as examples of subjective developer experiences.

This approach is intended to balance  
**transparency, privacy, and safety**.

## Japanese

# Developer Experience Observatory

実際のソフトウェア開発現場における  
「想定外の時間コスト」を観測・分析するための  
中立的なオブザーバトリー（観測所）です。

---

## 目的（Purpose）

**Developer Experience Observatory** は、実際のソフトウェア開発現場において
開発者が直面している **予期せぬ時間コスト** を収集・分析するための、
中立的な観測プロジェクトです。

本プロジェクトの目的は、特定の製品や組織を評価・批判することではなく、
開発者の時間、学習コスト、生産性に影響を与える **構造的なパターン** を
スケールの観点から観測することにあります。

この取り組みは、時間の経過とともに、
UI を学び直すことや頻繁な仕様変更への対応といった、
価値創出に直接つながらない付随的な時間コストを
減らすことに寄与することを目指しています。

こうした観測は重要です。
なぜなら、開発環境における予期せぬ変動性は、
長期的には大きく、しかも見えにくい経済的損失と
結びついているからです。

---

## 本プロジェクトが「すること / しないこと」

### すること

- 観測データに基づく **調査・分析プロジェクト**
- 以下のような要因による「想定外の時間損失」の記録：
  - 予告のない変更、または不明瞭な変更
  - ドキュメント更新の遅れ
  - ツール間の不整合
- 開発者体験改善に向けた **建設的な議論を支えるデータ提供**

### しないこと

- バグトラッカー
- 個別企業への問い合わせ・苦情窓口
- 製品・組織のランキングや格付け
- 個人・チーム・企業・製品を
  非難・攻撃・名指しする行為

---

## プライバシーと匿名性

- すべての投稿は **匿名** で行われます
- アカウント登録や SNS ログインは不要です
- 個人を特定する情報は意図的に収集しません
- ボット等の自動投稿対策を実施しています

本プロジェクトは、
**必要最小限のデータ収集**を原則としています。

---

## データ収集方針

### 収集する情報

- プラットフォーム（例：iOS、Web、Android）
- ツール／サービスのカテゴリ
- 変更の種類（予告あり／なし／不明）
- 想定外にかかった時間（レンジ形式）
- チーム規模（レンジ形式）

自由記述欄は文字数を制限し、
集計・傾向把握の補助目的のみに使用します。

### 収集しない情報

- 個人名
- アカウント情報
- 社内情報
- ソースコードや機密情報

---

## データの限界と適用範囲

本プロジェクトでは、
**個々の投稿内容の正確性や真偽を検証しません**。
投稿は、開発者個人の **主観的な体験**として扱われます。

また、本データは特定の企業・製品・組織を
評価・比較・判断する目的のものではありません。
あくまで「想定外の時間コストの傾向」を
観測するためのものです。

相関関係は因果関係を意味するものではありません。
構造的な解釈は仮説として提示されます。

---

## データ公開方針

- 個別の生データは公開しません
- 集計済みデータを定期的に公開します（例：月次）
- CSV や可視化資料の形式で提供します
- 分析手法やコードは公開します

個別投稿は、
個人や組織を特定しない形で、
**例示目的**として言及される場合があります
（例：「ある開発者からは次のような報告がありました」）。

これらは検証済みの事実ではなく、
主観的体験の一例として扱われます。
